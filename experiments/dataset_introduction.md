## 5. 測試資料

### 5.1 Soundscape 格式

測試資料為野外錄音的 soundscape，與訓練資料的單一物種錄音有顯著差異：

| 特徵 | 訓練資料 | 測試資料 |
|------|----------|----------|
| 內容 | 單一物種叫聲 | 野外環境錄音 |
| 背景噪音 | 少 | 多（風聲、水聲、其他動物） |
| 物種數量 | 通常 1 種 | 可能 0-多種同時存在 |
| 錄音品質 | 相對乾淨 | 真實環境，品質參差 |

### 5.2 提交格式

- **Row ID 格式：** `BC2026_Test_0001_S05_20250227_010002_5`
  - `BC2026_Test_0001`：soundscape 檔名
  - `S05`：感測器編號
  - `20250227_010002`：時間戳記
  - `5`：片段結束時間（秒）

- **輸出：** 每個 row_id 對應 234 個物種的存在機率 `[0, 1]`
- **Baseline：** 均勻分布 `1/234 ≈ 0.004274`

## 6. 挑戰與特點

### 6.1 主要挑戰

1. **Domain Gap：** 訓練資料（乾淨單一叫聲）vs 測試資料（嘈雜 soundscape）
2. **長尾分布：** 18 個物種 ≤ 5 筆錄音，模型難以學習稀少種類
3. **品質不一：** 36% 錄音未評分，品質參差不齊
4. **多標籤複雜性：** 12% 錄音包含多個物種，標註可能不完整
5. **地理多樣性：** 全球範圍，不同地區的同種鳥類叫聲可能有差異

### 6.2 資料集優勢

1. **規模大：** 35,549 筆錄音，344.5 小時音訊
2. **多樣性：** 涵蓋 5 個分類群，234 個物種
3. **真實性：** 來自公民科學和專業資料庫的真實野外錄音
4. **標準化：** 統一的 32kHz .ogg 格式
5. **地理覆蓋：** 全球範圍的生物聲學資料

## 7. 資料使用建議

### 7.1 訓練策略

- 使用 `rating_threshold` 過濾低品質錄音（建議 ≥ 1.0）
- 採用 `WeightedRandomSampler` 處理長尾分布
- 加入背景噪音增強縮小 domain gap
- 利用 secondary_labels 進行多標籤學習

### 7.2 驗證策略

- 使用 stratified split 確保每個物種在 train/val 都有代表
- 考慮用 soundscape 資料做 validation（如果有 train_soundscapes_labels.csv）
- 注意 val_roc_auc 可能虛高，以 Kaggle public score 為準

### 7.3 推論策略

- 使用 overlapping windows（hop < segment_duration）提升召回率
- 考慮 TTA（Test Time Augmentation）增加穩定性
- 對稀少種類可能需要調整閾值或後處理
