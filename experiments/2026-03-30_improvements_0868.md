# 2026-03-30: 改進紀錄 - 達成 0.868 分數

## 實驗結果
- **分數**: 0.868 (從 0.858 提升)
- **日期**: 2026-03-30
- **模型**: EfficientNet-B0 + 64ch CNN denoiser

## 實施的改進

### 1. 增加 Mel 頻譜圖解析度
- **變更**: `n_mels: 128 → 160`
- **原因**: 朋友的模型使用 160 mel bins，提供更細緻的頻率解析度
- **影響**: 能捕捉更多鳥類聲音的頻率細節

### 2. Secondary Labels 軟標籤
- **變更**: 次要標籤從 `1.0` 改為 `0.3`
- **位置**: `birdclef2026/src/dataset.py` 的 `_make_label_vector()`
- **原因**: 次要標籤的確定性較低，使用軟標籤可減少標籤噪音
- **實作**:
```python
vec[self._label_map[primary_label]] = 1.0  # 主要標籤
vec[self._label_map[lbl]] = 0.3             # 次要標籤
```

### 3. Mean + Max Pooling
- **變更**: 從單一 `avg pool` 改為 `mean_max` pooling
- **位置**: `birdclef2026/src/model.py`
- **原因**: 同時捕捉平均特徵和最顯著特徵，特徵維度從 1280 → 2560
- **實作**:
```python
mean_pool = features.mean(dim=[2, 3])  # 全局平均
max_pool  = features.amax(dim=[2, 3])  # 全局最大
pooled = torch.cat([mean_pool, max_pool], dim=1)
```

### 4. Gradient Clipping
- **變更**: 加入梯度裁剪 `max_norm=5.0`
- **位置**: `birdclef2026/src/train.py`
- **原因**: 防止梯度爆炸，穩定訓練過程
- **實作**:
```python
torch.nn.utils.clip_grad_norm_(model.parameters(), 5.0)
```

### 5. Early Stopping 指標改為 mAP
- **變更**: 從 `val_roc_auc` 改為 `val_map` (mean Average Precision)
- **位置**: `birdclef2026/src/train.py`
- **原因**: mAP 對於多標籤分類更敏感，更適合鳥類聲音辨識
- **實作**: 
  - 在 `evaluate.py` 加入 `macro_map` 計算
  - 訓練時追蹤 `best_val_map` 而非 `best_val_roc_auc`

### 6. Label Smoothing
- **變更**: 加入 `label_smoothing: 0.05`
- **位置**: `birdclef2026/config/local.yaml`
- **原因**: 防止模型過度自信，提升泛化能力
- **實作**:
```python
if label_smoothing > 0.0:
    batch_labels = batch_labels * (1.0 - label_smoothing)
```

## 下一步建議

### 短期改進 (預期提升 0.005-0.015)
1. **Auxiliary Loss (輔助損失)**
   - 加入 5 個分類群 (taxonomy) 的輔助分類任務
   - 權重 0.2，幫助模型學習鳥類的層級結構
   
2. **更強的數據增強**
   - 增加 Background Noise Mixing 的多樣性
   - 加入 Pitch Shifting (±2 semitones)
   - 加入 Time Stretching (0.9-1.1x)

3. **Test-Time Augmentation (TTA)**
   - 推理時使用多個 crops
   - 對預測結果取平均

### 中期改進 (預期提升 0.010-0.030)
4. **模型集成 (Ensemble)**
   - 訓練多個不同 seed 的模型
   - 訓練不同架構 (EfficientNet-B1, ResNet50)
   - 對預測結果加權平均

5. **更長的訓練時間**
   - 增加 `num_epochs` 到 300
   - 調整 learning rate schedule

6. **Focal Loss**
   - 替換 BCEWithLogitsLoss
   - 更關注難分類的樣本

### 長期改進 (預期提升 0.020-0.050)
7. **更大的模型**
   - EfficientNet-B2 或 B3
   - 需要更多 GPU 記憶體和訓練時間

8. **外部數據**
   - 使用 Xeno-Canto 的額外鳥類聲音
   - 需要仔細的數據清理和標籤對齊

9. **進階架構**
   - Transformer-based models (AST, BEATs)
   - 需要大量計算資源

## 已放棄的方法
- **PCEN**: 分數從 0.858 降到 0.477
  - 原因: 訓練集是短片段乾淨音頻，測試集是長片段環境音，PCEN 的 EMA 背景估計產生嚴重的 domain shift
- **128ch Denoiser**: 分數 0.850 (比 64ch 的 0.858 低)
  - 原因: 過擬合到 train_audio 的乾淨音頻，無法泛化到測試集
- **TRA (Temporal Recurrent Attention)**: 分數 0.818
  - 原因: 5 秒片段太短，時序注意力機制無法發揮作用
